Modal and Quick Start Direction

Three implementation directions for the first-run quick start, credential completion dialog, and compact modal refresh.

A. Unified Compact Modal Shell

Recommended. One shared modal foundation, all overlays clipped inside the content contour, quick start uses the same dense pattern.

B. Targeted Patch

Patch each existing dialog in place. Lower immediate churn, but duplicated spacing, focus, and overlay behavior are likely to drift again.

C. Separate Native Modal Route

Push more dialogs into dedicated modal windows. Good for minimal mode, heavier for advanced mode and less consistent with in-window flows.

Proposed Technical Spine

Core validates and initializes config at startup, emits a quick-start request only when first-run repair happened, and the renderer only presents the request. Credential completion becomes a single form that asks for missing username and/or password, with remember-password synced to settings when confirmed.

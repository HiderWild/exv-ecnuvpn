## Findings

### 1. src/app_api.cpp — Routing and orchestration (PASS)

The app API file routes requests to platform adapters. It does not contain platform-specific logic inline. VPN connect/disconnect delegates to helper or direct/elevated paths based on platform availability.

### 2. src/helper.cpp / src/helper.hpp — Orchestration (PASS)

Helper lifecycle orchestration delegates to platform-specific implementations via:
- `helper_lifecycle.hpp` → `src/platform/{darwin,win32,linux}/helper_lifecycle.cpp`
- `helper_service_manager.hpp` → `src/platform/{darwin,win32,linux}/helper_service_manager.cpp`
- `helper_platform.hpp` → `src/platform/{darwin,win32,linux}/helper_platform.cpp`
- `helper_client.hpp` → `src/platform/{darwin,win32,linux}/helper_client.cpp`

No platform-specific #ifdef guards in helper.cpp — all platform behavior is delegated via the adapter pattern.

### 3. src/helper_daemon_win.cpp — Windows dedicated service executable (PASS)

Uses dedicated helper service executable with named-pipe behavior. The daemon entrypoint is Windows-specific and correctly located in the top-level src/ (not in platform/).

### 4. src/helper_service_win.cpp — Windows helper service (PASS)

Windows service management (install, uninstall, start, stop) via SCM. Correctly uses Windows-specific APIs.

### 5. src/vpn.hpp / src/vpn_runtime.cpp — VPN runtime delegation (PASS)

VPN runtime delegates to platform adapters for:
- Process control → `src/platform/{darwin,win32}/process_control.cpp`
- OpenConnect process management → `src/platform/{darwin,win32}/openconnect_process.cpp`
- VPN supervisor → `src/platform/{darwin,win32}/vpn_supervisor_process.cpp`

The runtime orchestrates connect/disconnect but does not contain platform-specific tunnel or process logic.

### 6. macOS helper paths — Verified (PASS)

- Helper-installed path: launchd helper via `helper_service_manager.cpp` (install/uninstall)
- Helper-missing path: direct elevated session via `helper_lifecycle.cpp` (copy_self_to_stable_path_and_reexec, spawn_worker_process)
- Disconnect privilege path: `process_control.cpp` handles elevated disconnect correctly

### 7. Windows helper path — Verified (PASS)

- Helper-installed: named-pipe IPC via `helper_client.cpp`
- Service executable: `helper_daemon_win.cpp` + `helper_service_win.cpp`
- SCM-based install/uninstall
